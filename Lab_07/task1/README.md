# Task 1: Singleton Pattern
## Централизованное управление настройками

### Общее описание

В данной задаче реализован паттерн Singleton для создания менеджера конфигурации.  
Цель — гарантировать, что в приложении существует только один экземпляр класса, независимо от способа его вызова.

### Основные задачи

- Создать класс ConfigManager по принципу Singleton
- Исключить возможность создания нескольких экземпляров
- Проверить работу через разные способы инициализации
- Сравнить реализацию через класс и через ES-модуль

---

## Структура решения

### singleton.js
Классическая реализация Singleton:
- Проверка существующего экземпляра в конструкторе
- Статический метод `getInstance()`
- Методы `get()`, `set()`, `getAll()`

### config.js
Модульный вариант Singleton:
- Использование ES-модуля
- Экспорт единственного объекта

### test.js
Проверка:
- Создание через `getInstance()`
- Создание через `new`
- Проверка равенства ссылок
- Проверка сохранения данных

---

## Ключевые характеристики

| Критерий | Решение |
|-----------|----------|
| Инициализация | Ленивая (при первом обращении) |
| Хранение экземпляра | Статическое свойство |
| Защита от повторного создания | Проверка в конструкторе |
| Доступ к данным | Через публичные методы |
