## 2. Конкретные промпты

### Промпт 1:
`Объясни разницу между static import и dynamic import. Когда выгодно использовать import()?`
**Что получил:** Пояснение, что статический импорт грузится сразу, а динамический — по событию/условию, что уменьшает начальную загрузку.

### Промпт 2:
`Покажи пример dynamic import при клике по кнопке (Import on Interaction) для подключения расширенных функций.`
**Что получил:** Пример обработчика кнопки с `await import(...)` и вызовом функций из модуля.

### Промпт 3:
`Как сделать ленивую загрузку модуля при появлении блока на экране через IntersectionObserver?`
**Что получил:** Шаблон с IntersectionObserver, проверкой `isIntersecting` и подгрузкой модуля истории.

### Промпт 4:
`Как правильно обработать ошибку при dynamic import (try/catch) чтобы приложение не падало?`
**Что получил:** Рекомендации оборачивать импорты в `try...catch`, выводить ошибку в консоль и показывать понятное сообщение пользователю.

## 3. Модификация и проверка
- Сам спроектировал разделение модулей: базовые (constants/core/utils) как static, расширенные (advanced/history) как dynamic.
- Реализовал `app.mjs` как “точку сборки”, которая подключает модули и управляет состоянием калькулятора.
- Настроил lazy load истории через IntersectionObserver, чтобы модуль грузился только при скролле к блоку.
- Тестировал в браузере через локальный сервер (`npx serve .`), чтобы модули корректно импортировались без проблем CORS.

## 4. Что узнал
- Чем отличается eager загрузка от lazy и как это влияет на скорость первого запуска.
- Как использовать `import()` для подгрузки функционала “по требованию”.
- Как работает `IntersectionObserver` и где он полезен (ленивая загрузка блоков).
- Как проектировать модульную архитектуру так, чтобы новые возможности добавлялись как отдельные файлы без переписывания ядра.
