1. Сначала я определяла основную проблему, которую решает каждый фрагмент кода:
   - Контроль создания объектов → Creational
   - Композиция, обёртка, скрытие сложности, совместное использование данных → Structural
   - Взаимодействие, уведомления, координация действий → Behavioral

2. Затем я определяла конкретный паттерн по характерным признакам:
   - Singleton — хранение единственного экземпляра и глобальная точка доступа.
   - Factory Method — создание объектов через функцию/метод, а не напрямую через new.
   - Facade — единый класс, скрывающий сложность подсистемы.
   - Decorator — добавление поведения через оборачивание объекта.
   - Prototype — создание объектов через клонирование.
   - Observer — механизм подписки и уведомления.
   - Chain of Responsibility — передача запроса по цепочке обработчиков.
   - Command — инкапсуляция действия в объект с методами execute/undo.
   - Flyweight — разделение внутреннего (общего) и внешнего (уникального) состояния.
   - Mediator — центральный координатор взаимодействия объектов.
