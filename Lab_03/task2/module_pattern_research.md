Исторический контекст и эволюцию паттерна Module
    Module Pattern появилась в JS из-за того что раньше не было нормальных модулей, и все легко улетало в глобальную область винды. Чтобы не перегрузить глобальные переменные, начали делать модуль через функцию + замыкание, чтобы скрывать внутренние данные и отдавать наружу только нужное.

Связь с другими паттернами (например, Revealing Module Pattern)
    Revealing Module Pattern похожий, но считается больее читабельным, потому что ты расскрываешь что именно отдаешь наружу

Известные случаи использования в популярных JavaScript-библиотеках и фреймворках:
    В старых библиотеках\коде до ES6; в коде где важно скрыть внутреннюю логику и оставить API

Вариации и современные адаптации (ES6-модули)
    Обычный модуль; модуль с приватными переменными; revealing module
