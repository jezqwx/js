# Lab 4.2: Модернизация legacy JavaScript-приложения

## Описание работы

В рамках лабораторной работы выполнен рефакторинг устаревшего кода интернет-магазина.  
Основная цель — преобразовать монолитную реализацию с глобальными переменными в структурированную модульную архитектуру без изменения бизнес-логики приложения.

## Структура проекта

- `legacy_code.js` — оригинальная версия приложения с антипаттернами  
- `refactoring_plan.md` — анализ проблем и выбранная стратегия изменений  
- `refactoring_documentation.md` — подробное описание проведённых изменений  
- `refactored_modules/` — итоговая модульная реализация  

## Использованные архитектурные подходы

- **Namespace Pattern** — объединение всей логики под одним глобальным объектом `MyApp`
- **Module Pattern (IIFE)** — создание приватных областей видимости
- **Revealing Module Pattern** — формирование понятного публичного интерфейса модулей

## Основные изменения

- Удалены глобальные переменные
- Бизнес-логика разделена на независимые сервисы
- Настройки вынесены в отдельный конфигурационный модуль
- Дублирующиеся функции объединены
- Работа с localStorage вынесена в отдельный слой
- Реализована инкапсуляция состояния корзины, товаров и пользователя

## Архитектурные различия

| Критерий | Legacy-версия | Рефакторинг |
|-----------|--------------|-------------|
| Глобальная область | Множество переменных | Один объект `MyApp` |
| Организация кода | Неструктурированный монолит | Разделение на логические модули |
| Инкапсуляция | Отсутствует | Реализована через IIFE |
| Расширяемость | Сложная | Упрощена |
| Управление конфигурацией | Разбросано по коду | Централизовано |

## Итог

В результате работы код стал более структурированным и масштабируемым.  
При этом сохранена исходная функциональность интернет-магазина (расчёт налогов, доставка, обработка заказов и работа с хранилищем).

Рефакторинг позволил устранить основные антипаттерны и снизить технический долг проекта.
