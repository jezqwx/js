# Документация процесса рефакторинга — Лабораторная работа 4.2

## 1. Что было изменено и зачем
- **Убрано глобальное состояние:** массивы `cart`, `products`, объект `user` перенесены в приватную область внутри `MyApp.Services`.
- **Вынесены настройки:** все значения (налог, доставка, валюта, язык, тема, скидка) перенесены в `MyApp.Config` и остались как в legacy.
- **Отделено хранилище:** работа с `localStorage` вынесена в `MyApp.Storage`, ключи сохранены без изменений (`" products "`, `" cart "` и т.д.).
- **Убрано дублирование:** subtotal/total/format объединены в единые функции.

## 2. Как Module Pattern решает проблемы (Гл. 7)
- **Инкапсуляция:** `_cart`, `_products`, `_user` скрыты внутри IIFE. Их нельзя менять напрямую из консоли.
- **Контроль доступа:** внешнему коду доступны только методы сервисов (например, `CartService.add`, `OrderService.process`).
- **Упрощение тестирования:** можно тестировать сервисы отдельно, не трогая глобальные переменные.

## 3. Как Namespace Pattern решает проблемы (Гл. 11)
- В глобальной области есть только `MyApp`.
- Модули организованы по зонам ответственности: `MyApp.Config`, `MyApp.Utils`, `MyApp.Storage`, `MyApp.Services`.
- Риск конфликтов имен с другими скриптами уменьшается.

## 4. Что сохранилось (важно для лабораторной)
- Формулы расчёта налогов/доставки/итога
- Порог бесплатной доставки (`> 50`)
- Значения по умолчанию (`shippingCost=5`, `taxRate=0.08`, и т.д.)
- Формат order id: `" ORD - " + Date.now()`
- Ключи localStorage (с пробелами)

## 5. Итог
Код стал модульным, читабельным и безопасным, при этом полностью повторяет поведение исходной реализации.
