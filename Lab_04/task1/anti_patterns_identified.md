# Найденные архитектурные проблемы

## 1. Глобальные переменные

**Проблема**: Использовать var в глобальной области.

**Последствия**: Любая часть кода может изменить данные.
**Пример**:
```javascript
var userName = "";
var userEmail = "";
var isLoggedIn = false;
```
---

## 2. Повторяющаяся логика валидации

**Проблема:** Несколько функций выполняют одинаковую проверку email.

**Последствие:** Повышается риск рассинхронизации логики.

---

## 3. Конфигурация вне структуры

**Проблема:** Настройки API и таймауты объявлены напрямую в коде.

**Последствие:** Сложность изменения параметров.

---

## 4. Глобальные event handlers

**Проблема:** onLoginClick и onLogoutClick доступны глобально.

**Последствие:** Повышенный риск конфликтов имен.

---

## 5. Сильная связность обработки данных

**Проблема:** processUserData управляет всем сразу.

**Последствие:** Невозможно протестировать логику изолированно.
