# Лабораторная 1.2: Реализация Provider Pattern

# Документация
В этой лабораторной работе был реализован паттерн Provider с использованием React Context Api. Основная проблема, которую решает Provider Pattern это prop drilling. Prop drilling это ситуация, когда данные приходится передавать через много уровней компонентов, даже если промежуточные компоненты эти данные не используют.
В это приложении нескольким вложенным компонентом нужный данные пользователя, такие как имя, почта, роль, права доступа, и настройки темы. Без Provider Pattern эти данные пришлось бы передавать из App в dashboard, затем в Header, потом в UserMenu, что усложняет код.
Для решения этой проблемы был создан UserContext и компонент UserProvider, который хранить данные пользователя в одном месте. Все вложенные компаненты получают доступ к этим данным напрямую через хук useUser, без передачи props.
Плюсы данного подхода: код становится чище и понятнее; умнеьшится количество пропс; данные удобно хронить и изменять в одном месте.
